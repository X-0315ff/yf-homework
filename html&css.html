3.掌握动画:

动画说明：
动画是网页制作中常见的一种效果，通过改变元素的位置、大小、颜色、透明度等属性来创建一种动态的效果。
动画分为两种：
一.CSS3动画：通过使用CSS3的动画属性来创建动画效果。
其中css3有过渡和动画两种方式实现动画：
过渡：通过设置元素的属性值来创建动画效果，例如：
transition-property：定义要过渡的属性。
transition-duration：定义动画的持续时间。
transition-timing-function：定义动画的速度曲线。
transition-delay：定义动画的延迟时间。
2. 动画：通过定义多个关键帧来创建动画效果，例如：
定义动画
/* 方式一 */ @keyframes 动画名称 {   from {}   to {} } /* 方式二 */ @keyframes 动画名称 {   0% {}   10% {}   ......   100% {} }
使用动画： animation: 动画名称 动画花费时长;

二.JavaScript动画：通过使用JavaScript来控制元素的位置、大小、颜色、透明度等属性来创建动画效果。

3.css的重绘重排以及避免重绘重排:

重绘：指页面元素在视觉上发生变化，但内存中的元素没有发生变化。

重绘一定不重排

触发重绘的操作：

改变元素的颜色、背景色、文字颜色、透明度等。

修改 border 或 outline 样式。

设置 visibility（从 hidden 改为 visible，不改变位置和尺寸）。

改变 text-shadow 或 box-shadow 等视觉效果。

注意： 重绘不会改变页面的布局，所以它不会影响元素的几何属性，如位置、大小、边距等。尽管如此，重绘仍然是一个性能开销较大的操作，特别是当涉及到多个元素时。


重排：指页面元素在视觉上发生变化，并且内存中的元素也发生变化，任何会影响页面布局的操作都会触发重排。

重排一定重绘！

触发重排的操作：

改变元素的位置、大小、边距、边框、内边距等。

修改 display、position、float、z-index 等属性。

设置 visibility（从 visible 改为 hidden，不改变位置和尺寸）。

设置 overflow（从 hidden 改为 visible，不改变位置和尺寸）。

设置 perspective（不改变位置和尺寸）。

设置 perspective-origin（不改变位置和尺寸）。

注意： 重排会导致浏览器重新计算元素的 位置 和 尺寸，然后重新渲染整个页面的布局。重排通常比重绘要更消耗性能，因为它不仅需要重新绘制页面的元素，还需要计算和重新布局所有受影响的元素。


如何避免重绘和重排？

1. 批量操作 DOM 元素：如果你需要同时改变多个 DOM 元素，最好将它们的修改操作放在一起，避免频繁触发重排和重绘

2. 使用 visibility 替代 display ：因为visibility：hidden 是隐藏元素，并占位置，不可交互，而display:none,会完全移除元素，导致重排

3. 尽量避免直接修改布局相关属性: 如需要频繁更新元素位置或尺寸，考虑使用 transform 属性代替 top、left、width、height 等。 因为 transform 与布局（layout）无关，当你使用 transform: translate() 来移动元素时，元素的实际位置和尺寸并没有发生变化， 它只是 在屏幕上 移动或改变了外观，浏览器不需要重新计算整个页面的布局。

4. 避免在循环中频繁操作 DOM： 如果在 JavaScript 中有循环，避免在每次循环中直接操作 DOM。最好在循环结束后一次性更新 DOM。

4.flex布局与grid布局:

flex布局：是一种用于 一维布局（单行或单列）的 CSS 布局方式。它能让子元素自动调整大小，适应各种屏幕，并支持灵活的对齐和分布。

要使用 flex 布局，只需在父容器上设置： display: flex;这样，子元素（flex items）会按照主轴排列。

属性              作用
display: flex; 开启弹性布局
flex-direction 控制主轴方向
justify-content 控制主轴对齐方式
align-items 控制交叉轴对齐方式
flex-wrap 控制是否换行
align-content 控制多行对齐方式
flex 子元素弹性伸缩
order 控制子元素顺序
align-self 控制单个子元素对齐

grid布局：是一种用于二维布局（多行多列）的 CSS 布局方式。它能让子元素自动调整大小，适应各种屏幕，并支持灵活的对齐和分布。

要使用 grid 布局，只需在父容器上设置： display: grid;这样，子元素（grid items）会按照网格进行排列。

grid-template-columns（定义列） ： 控制 列数 & 宽度

grid-template-rows（定义行）： 控制 行数 & 高度

grid-gap: 10px; 设置行列间距：行间距 10px，列间距 10px

grid-column（跨列） ：子元素可以跨多个列。 grid-column: span 2;跨 2 列

grid-row（跨行） ： 子元素可以跨多个行。grid-row: span 2; 跨 2 行

justify-content（水平对齐）：start 左对齐，end 右对齐，center 居中，space-between 两端对齐，space-around 四周留间距

align-content（垂直对齐）： 和水平对齐类似

5.css居中对齐方法:

text-align: center ==> 行内元素 ： text-align: center;

margin: auto; ==> 块级元素 ： margin: 0 auto;

line-height ==> 单行文本 ： line-height: 100px; 设置值与文本高度一致

text-align: center ==> 任意元素（需要子绝父相） ： top: 50%; left: 50%; transform: translate(-50%, -50%) 左上角顶点先移到中心点，再向左向右移动自身的一半

flexbox ==> 现代布局 ： display: flex; justify-content: center; align-items: center; 垂直和水平都居中

grid ==> 现代布局 ： display: grid; place-items: center; 垂直和水平都居中

6.绝对定位和相对定位，固定定位(加分项):

1.relative（相对定位）

特点：

相对于自身原来的位置进行偏移（不会脱离文档流）

不会影响其他元素的布局。

仍然占据原来的空间，只是视觉上移动。

常用于absolute的父级，给子元素 absolute 作为定位参照

2. absolute（绝对定位）

特点：

完全脱离文档流，不会占据原来的空间。

相对于最近的 position: relative/absolute/fixed 祖先元素进行定位，如果没有，默认相对于根元素定位（即 body）。

不影响其他元素的布局，元素像是漂浮在页面上。

3. fixed（固定定位）

特点：

完全脱离文档流，不会占据原来的空间。

相对于 viewport （浏览器窗口）进行定位，不随页面滚动而移动。

适用于固定导航栏、返回顶部按钮等。

7.脱离文档流

脱离文档流方法：

position: absolute;（绝对定位）

position: fixed;（固定定位）

float: left/right;（浮动）

position: sticky;（粘性定位）：sticky 定位的元素部分脱离文档流，它在一定范围内是普通流元素，但当滚动到设定的偏移位置时，会变为固定定位。

好处:回想之前学到的重绘与重排， 如果元素脱离文档流那么重排就影响不到这个元素，将经常触发重排的元素脱离文档流会大大提高性能

8.BFC（Block Formatting Context，块级格式化上下文）:

BFC是一个 CSS 概念，描述了元素如何在页面中布局和排列的方式。 它是一个容器，用于控制元素的布局规则，特别是如何处理子元素的浮动、外边距折叠、以及定位等属性。

当某个元素满足以下条件之一时，就会创建一个新的 BFC：

元素的 display 值为 block、inline-block、flex、grid、table 等。

元素的 float 值为 left 或 right。

元素的 position 值为 absolute 或 fixed。

元素的 overflow 值为 auto、scroll 或 hidden。

BFC 的应用场景：

清除浮动 ： 浮动元素会脱离文档流，导致父元素高度塌陷（无法撑开高度）。如果想让父元素包含浮动元素，可以通过创建 BFC 来解决。

例如：a1子元素是浮动元素，脱离文档流导致父元素A高度塌陷，如果想让父元素A包含浮动元素a1，可以通过创建 BFC 来解决：overflow: hidden;

避免外边距合并 ： 当两个块级元素的外边距接触时，它们的外边距会合并为最大值。通过给父元素创建 BFC（overflow: hidden;），子元素可以防止这种外边距合并。

防止浮动元素影响页面布局 ： 如果页面中存在浮动元素，而不希望它影响到页面其他内容的布局（例如浮动元素覆盖其他内容），可以通过使用 BFC（overflow: hidden; ） 来解决。
